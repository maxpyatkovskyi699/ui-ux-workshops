# ПЗ-12 Розробка wireframe

## **Мета роботи**

- Створити у Figma деталізований вайрфрейм інтернет-магазину (головна сторінка та 2–3 сторінки товару) з адаптацією під розмір екранів desktop.
- Використати інструменти прототипування (Interactions) для побудови клікабельного прототипу на основі розробленого вайрфрейму.

## **Хід роботи**

1. **Вибір тематики магазину**  
   Обрано тематику інтернет-магазину Prom.ua.

2. **Створення структури сайту**
   Структура макету включає такі сторінки:

   - Головна сторінка
   - Сторінка товару №1
   - Сторінка товару №2
   - Сторінка товару №3
   - Сторінка товару №4
   - Сторінка товару №5

3. **Розробка вайрфреймів у Figma**

   - Створено desktop-фрейми шириною 1280 px.
   - Додано основні елементи інтерфейсу:
     - хедер
     - футер
     - глобальне меню
     - карточки товарів
     - банери
     - попередні перегляди товарів
   - Всі елементи виконано у стилі середньої деталізації (_medium-fidelity_), із чіткою структурою та логічним розташуванням компонентів.

4. **Налаштування взаємодій (Interactions)**
   Реалізовано такі типи інтеракцій:

   - **On Click → Navigate To** — переходи між сторінками прототипу.
   - **On Hover → Change Style (Button Hover)** — затемнення кнопки при наведенні курсора.

5. **Створення клікабельного прототипу**
   - Налаштовано зв’язки між усіма основними екранами.
   - Додано навігацію через кліки по карточках товарів та кнопках.
   - Прототип протестовано в режимі **Present**.

## **Результат виконання**

У результаті виконання роботи було створено:

- деталізований вайрфрейм інтернет-магазину у Figma;
- головну сторінку та п’ять сторінок товарів;
- коректну навігацію між екранами;
- інтерактивний прототип із використанням інструментів Interactions;
- демонстрацію базової логіки роботи інтернет-магазину на рівні UX-компонування.

## **Висновки**

У процесі виконання роботи я ознайомився з методологією створення вайрфреймів та принципами побудови структурної логіки інтерфейсу.  
У Figma були використані інструменти прототипування для реалізації інтерактивних сценаріїв переходів.  
Створений прототип демонструє послідовність взаємодії користувача з елементами інтерфейсу та може слугувати основою для подальшої розробки високодеталізованого дизайну.
